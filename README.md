## 功能特点

- ⏰ **智能提醒** - 在会话过期前自动发送提醒（5天、3天、1天）
- 🔇 **静默时段** - 支持设置静默时段，避免半夜打扰
- 📱 **提前获取二维码** - 在会话过期前获取新登录二维码，无缝续期
- 📲 **多渠道发送** - 支持同时发送到 Telegram 和微信
- 🔄 **自动检测** - 自动检测登录成功事件并更新登录时间
- 💬 **双端命令支持** - 支持在 Telegram 和微信发送命令
- 📊 **状态查询** - 随时查看所有监控频道的会话状态
- ⚙️ **灵活配置** - 支持自定义所有参数

## 背景

微信网页版登录有效期通常为 **30天**，过期后需要重新扫码登录。如果会话在半夜过期，可能会导致：

- ❌ 错过重要消息
- ❌ 需要半夜起来重新登录
- ❌ 消息收发中断

本中间件通过以下方式解决这些问题：

```
┌─────────────────────────────────────────────────────────────┐
│                    解决方案                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1️⃣ 提前提醒（5天、3天、1天）                             │
│     ↓                                                       │
│  2️⃣ 静默时段（凌晨不打扰）                                 │
│     ↓                                                       │
│  3️⃣ 提前获取二维码（会话未过期时）                         │
│     ↓                                                       │
│  4️⃣ 微信/Telegram 双端提醒                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 安装

### 方法一：通过 pip 安装（推荐）

```bash
pip install efb-session-reminder-middleware
```

### 方法二：从源码安装

```bash
git clone https://github.com/chern91552/efb-session-reminder-middleware.git
cd efb-session-reminder-middleware
pip install .
```

### 依赖要求

- Python 3.7+
- ehforwarderbot
- pyqrcode（用于生成二维码）

```bash
pip install pyqrcode
```

## 配置

### 1. 在 EFB 配置文件中启用中间件

编辑 `~/.ehforwarderbot/profiles/default/config.yaml`：

```yaml
master_channel: blueset.telegram
slave_channels:
  - blueset.wechat
middlewares:
  - efb_session_reminder  # 添加此行
```

### 2. 配置中间件

创建 `~/.ehforwarderbot/profiles/default/efb_session_reminder/config.yaml`：

```yaml
# ============================================
# 基础设置
# ============================================

# 是否启用中间件
enabled: true

# 会话有效期（天数），微信网页版通常为30天
session_validity_days: 30

# ============================================
# 提醒设置
# ============================================

# 提醒阈值（过期前几天发送提醒）
reminder_thresholds:
  - 5   # 5天前提醒
  - 3   # 3天前提醒
  - 1   # 1天前提醒

# 静默时段（不发送提醒的时间）
# 格式：[开始小时, 结束小时]（24小时制）
quiet_hours:
  - 0   # 凌晨 0:00
  - 8   # 到早上 8:00

# 检查间隔（秒），多久检查一次会话状态
check_interval: 3600  # 每小时检查一次

# ============================================
# 二维码设置
# ============================================

# 二维码生成阈值（天）
# 当剩余天数 <= 此值时，提醒中会包含二维码
qr_threshold_days: 3

# 是否在提醒中自动生成二维码
auto_qr: true

# ============================================
# 发送渠道设置
# ============================================

# 消息发送渠道
# 选项：'telegram'（仅Telegram）
#       'wechat'（仅微信）
#       ['telegram', 'wechat']（两者都发送）
delivery_channels:
  - telegram
  - wechat

# 微信接收人设置
# 选项：
#   - 'filehelper': 发送到文件传输助手
#   - 联系人名称: 发送到指定的联系人
#   - 群聊名称: 发送到指定的群聊
wechat_recipient: 'filehelper'

# ============================================
# 监控设置
# ============================================

# 监控的频道列表
monitored_channels:
  - blueset.wechat
```

## 使用

### 自动提醒

中间件会自动在以下时间点发送提醒：

| 剩余天数 | 提醒级别 | 示例消息 |
|---------|---------|---------|
| > 5天 | 正常 | 🟢 无需提醒 |
| 5天 | 提醒 | 🟢 5天后会话将过期 |
| 3天 | 警告 | 🟡 3天后会话将过期 |
| 1天 | 紧急 | 🟡 明天会话将过期 |
| < 1天 | 紧急 | 🔴 会话即将过期！ |

### 命令使用

#### 在 Telegram 中发送

| 命令 | 说明 |
|------|------|
| `getqr` 或 `qr` 或 `获取二维码` | 获取登录二维码 |
| `session` 或 `会话状态` | 查看会话状态报告 |
| `setlogintime <频道ID>` | 手动设置登录时间 |

#### 在微信中发送

向「文件传输助手」发送同样的命令：

| 命令 | 说明 |
|------|------|
| `getqr` 或 `qr` 或 `获取二维码` | 获取登录二维码 |
| `session` 或 `会话状态` | 查看会话状态报告 |

### 二维码登录流程

```
┌─────────────────────────────────────────────────────────────┐
│                  二维码登录流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1️⃣ 收到过期提醒 + 二维码                                   │
│     ↓                                                       │
│  2️⃣ 打开微信扫描二维码                                      │
│     ↓                                                       │
│  3️⃣ 在微信中点击「登录」                                   │
│     ↓                                                       │
│  4️⃣ 新会话替换旧会话，消息不中断                           │
│     ↓                                                       │
│  5️⃣ 系统自动检测登录，更新登录时间                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**重要**：扫码后，**原会话不会立即退出**，只有确认登录后才会切换。

## 常见问题

### Q: 二维码会过期吗？

A: 是的，二维码通常在 2-5 分钟后过期。如果过期了，发送 `getqr` 命令重新获取即可。

### Q: 获取二维码会影响当前会话吗？

A: **不会**。生成二维码只是在服务器端创建待登录凭证，原会话完全不受影响。只有扫码并确认登录后，才会切换到新会话。

### Q: 提醒会发送到微信吗？

A: 默认会同时发送到 Telegram 和微信。如果只想发送到某个渠道，在配置文件中修改 `delivery_channels`。

### Q: 如何更改微信接收人？

A: 在配置文件中修改 `wechat_recipient`：
- `filehelper` - 文件传输助手
- `张三` - 发送给联系人"张三"
- `工作群` - 发送给群聊"工作群"

### Q: 如何手动更新登录时间？

A: 发送 `setlogintime blueset.wechat` 命令，或者在微信频道重新登录。

### Q: 如何关闭半夜的提醒？

A: 在配置文件中设置 `quiet_hours`，例如 `[0, 8]` 表示凌晨0点到8点不发送提醒。

### Q: 支持监控多个频道吗？

A: 支持，在配置文件的 `monitored_channels` 列表中添加多个频道ID即可。

### Q: 如何在微信中使用命令？

A: 打开微信「文件传输助手」，发送命令即可。支持 `getqr`、`session` 等命令。

## 配置文件说明

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `enabled` | bool | `true` | 是否启用 |
| `session_validity_days` | int | `30` | 会话有效期（天） |
| `reminder_thresholds` | list | `[5, 3, 1]` | 提醒阈值 |
| `quiet_hours` | list | `[0, 8]` | 静默时段 |
| `check_interval` | int | `3600` | 检查间隔（秒） |
| `qr_threshold_days` | int | `3` | 二维码生成阈值 |
| `auto_qr` | bool | `true` | 自动生成二维码 |
| `delivery_channels` | list | `[telegram, wechat]` | 发送渠道 |
| `wechat_recipient` | str | `filehelper` | 微信接收人 |
| `monitored_channels` | list | `[blueset.wechat]` | 监控频道列表 |

### monitored_channels 详解

`monitored_channels` 用于指定要监控哪些从频道的登录状态。

```yaml
# 监控一个频道（默认配置，适合大多数用户）
monitored_channels:
  - blueset.wechat

# 监控多个频道（高级用户）
monitored_channels:
  - blueset.wechat        # 微信频道
  - another.wechat         # 第二个微信账号
  - some.other.slave       # 其他从频道
```

**普通用户保持默认即可**，只有在使用多个从频道时才需要修改。
